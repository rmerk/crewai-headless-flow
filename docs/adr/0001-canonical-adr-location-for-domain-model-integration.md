# Canonical ADR location is `docs/adr/`, layered as a prompt addendum rather than a skill swap

## Status
Superseded by ADR-0002 (OpenWiki pivot), then by ADR-0011 (Domain Model Integration rejected entirely). Kept for historical context. The `docs/adr/` location convention for *this* repository's own ADRs remains valid independently of the abandoned target-repo DMI feature.

The vendored `documentation-and-adrs` skill (used by the `finalize` stage) and the vendored `domain-modeling` skill (used by Domain Model Integration) prescribe two different ADR conventions: `docs/decisions/` with a fuller template vs. `docs/adr/` with a lean template. We can't edit either vendored skill in place. We decided `docs/adr/` with the lean template is canonical for this project, enforced via a prompt-level addendum injected alongside `documentation-and-adrs` when Domain Model Integration is enabled — not by swapping `finalize`'s skill to `domain-modeling`. This keeps `finalize` still benefiting from `documentation-and-adrs`'s README/changelog/inline-doc guidance, and keeps Domain Model Integration's effect on `plan`/`finalize` limited to context injection rather than skill resolution, which we'd already decided (see the `plan`-reads/`finalize`-writes shape of Domain Model Integration).
