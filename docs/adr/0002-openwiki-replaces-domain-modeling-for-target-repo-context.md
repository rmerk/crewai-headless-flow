# Domain Model Integration uses OpenWiki, not the `domain-modeling` skill's `CONTEXT.md`/`docs/adr/` convention

## Status
Accepted, supersedes the target-repo-context portion of the design in ADR-0001.

We initially designed Domain Model Integration around this project's own vendored `domain-modeling` skill: `plan` would read a target repo's `CONTEXT.md`, and `finalize` would write to `CONTEXT.md`/`docs/adr/` agent-driven, alongside its existing `documentation-and-adrs` work. We're replacing that entirely with [OpenWiki](https://github.com/langchain-ai/openwiki), a purpose-built third-party CLI that generates and maintains agent-facing documentation (`openwiki/`) and self-registers via the target repo's `AGENTS.md`/`CLAUDE.md`. This is a deliberate new kind of dependency for this project: OpenWiki is a separate Node/TypeScript CLI with its own inference provider and API key configuration (`~/.openwiki/.env`), invoked via subprocess, outside the `HeadlessCoder` adapter contract entirely. We accepted this the same way this project already accepts Codex/Grok/Claude/Gemini/Cursor CLI dependencies: installation and authentication are user-managed, not something the Flow provisions. This decision was made without a full build-vs-buy comparison against alternatives beyond OpenWiki; revisit if OpenWiki's maintenance/inference-cost tradeoffs prove unworkable in practice.
